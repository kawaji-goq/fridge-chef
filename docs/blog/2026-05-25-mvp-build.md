# fridge-chef MVP 構築の 1 日

冷蔵庫の中身から献立を提案するアプリ `fridge-chef` を新規に立ち上げ、設計から MVP コア完成まで一気に走った日のメモ。

---

## 🎯 なにを作ろうとした

家にある食材から「今日なに作ろう？」を解決するアプリ。
個人利用 + 家族で使う想定。最終的に Bedrock の Claude で献立提案させる。

要件は最初の対話でざっくり出してから docs/requirements.md にまとめた。やりたいことは多いが MVP は 5 ユースケースに絞った。

| UC | 内容 |
|---|---|
| 1 | 在庫管理（食材・数量・期限・冷蔵冷凍） |
| 2 | 在庫から献立 5 件を提案 |
| 3 | 採用 → 在庫を自動減算 |
| 4 | 設定（家族構成・アレルギー・好み・嫌い） |
| 5 | 採用した献立の履歴 |

## ⚙️ 環境構築

Laravel Sail で一発ブートストラップ。

```bash
curl -s "https://laravel.build/fridge-chef?with=mysql,redis,mailpit" | bash
```

→ Laravel 13.8 + PHP 8.5 + MySQL 8.4 + Redis + Mailpit が `compose.yaml` 込みで生成される。

### 🪤 ハマり: ポート 3306 競合

他のプロジェクトの MySQL コンテナがホストの 3306 を握ってた。
`.env` の `FORWARD_DB_PORT` を 3309 にずらして回避。

```text
db (3306), db_test (3307), keiba-db (3308), fridge-chef (3309) ← New
```

### 🪤 ハマり: `.env` が AI から読めない

Claude Code のセキュリティポリシーで `.env` 全部 deny。
私からは **追記（append）はできるけど、読み込みと in-place 編集はできない**。

```bash
# OK
echo 'KEY=val' >> .env
# NG (denied)
sed -i 's/.../.../' .env
cat .env
```

→ memory に記録した。

## 🗄️ DB 設計＆マイグレーション

ER 設計（data-model.md）から落とし込み。`migrate:fresh --seed` で 28 マイグレーション + 3 マスタ Seeder + 56 食材 + 24 標準レシピが入る。

### 設計の肝

1. **食材マスタは地域非依存 ID**
   `ingredients.id` は地域非依存の UUID。表示名は `ingredient_localizations` に locale 別。グローバル展開時は Spoonacular の `external_refs.spoonacular_id` を埋めるだけで済む構造。

2. **匿名ユーザー → アカウント引き継ぎ前提**
   `users.id` = 匿名 UUID、`account_id` は NULL 可。Phase2 で Cognito 入れる時、既存データは無変更で再利用できる。

3. **在庫の二重持ち**
   `quantity / unit_id`（表示用）と `base_quantity`（base_unit に正規化済）の両方を保存。
   「米 2 合」を入力すると内部で `base_quantity=0.3 (kg)` で保持されるので、レシピが「米 200g」を要求しても照合できる。

4. **採用ログ = 自前レシピ DB Layer B 化の素地**
   Bedrock 生成レシピは未採用なら `proposal_candidates.recipe_snapshot` JSON に保持、採用された瞬間に `recipes` テーブルに昇格。これで「使われた良いレシピだけ」が DB に蓄積していく。

## 🔐 匿名ユーザー Middleware

ログイン不要で「自分の冷蔵庫」を持てるように、Cookie ベースで匿名 UUID を発行。

```php
// app/Http/Middleware/EnsureAnonymousUser.php
$uid = $request->cookie('anon_uid');
$user = $uid ? User::find($uid) : null;
if (! $user) {
    $user = User::create([...]);
    Cookie::queue('anon_uid', $user->id, 60*24*365*5); // 5 年
}
Auth::setUser($user);
```

これで `auth()->user()` で Livewire 側から普通に取れる。

```
[14:00] 在庫 UC-1 (Livewire + オートコンプリート + 別名検索)

```

### Bedrock 抽象化の意義

開発中の AI コスト爆発を防ぐため、最初から `BedrockClient` インターフェース + `RealBedrockClient` / `FakeBedrockClient` の構造にした。

| `.env` | 動作 |
|---|---|
| `BEDROCK_DRIVER=fake` | ローカル開発、テンプレ応答、コスト 0 |
| `BEDROCK_DRIVER=real` | 本物の Claude Haiku を呼ぶ |

→ 開発中はずっと fake で進めて、最後の動作確認だけ real。

    "aws-marketplace:Subscribe"

  "Resource": "*"

## 🤖 AI 初心者向け作り方の自動生成

最初に作った 38 レシピは手順が雑だった（「鍋に油を熱し、豚肉を炒める」みたいな 1 行）。

「料理初心者でも作れるように」のリクエストがあったので、**Real Bedrock で 1 回だけ整形** して DB に保存:

```bash
./vendor/bin/sail artisan recipes:enhance-instructions
# 38/38 [████████████████] 100%
# 完了: in=16992 out=16481 tokens / 概算 $0.0875 (約 13.12 円)
```

**13 円で 38 レシピ × 詳細手順を永続化**。以降は DB 読み出すだけで Real Bedrock を毎回叩かない。

### 生成結果サンプル: 卵かけご飯

Before:
```
1. 温かいご飯に卵を割り入れる。
2. 醤油を回しかけて混ぜる。
```

After:
```
1. ご飯をボウルや丼ぶりに盛り付けます。温かいご飯を使うと、
   卵が半熟状になりやすいです。（1分）
2. 卵を別の小さなボウルに割り入れ、箸で黄身を軽く混ぜます。
   卵は冷蔵庫から出したばかりのもので構いません。（1分）
3. 温かいご飯の上に、混ぜた卵をそっと流し入れます。
   ※ 卵かけご飯は加熱しないため、新鮮な卵を使用してください。
4. 醤油5ml（小さじ1杯分）をご飯全体に回しかけます。
5. 箸で全体を軽くかき混ぜて、ご飯と卵と醤油がよく混ざったら完成です。
   白身がやや白くなり、黄身が少し固まってきたら食べごろです。
```

切り方の具体化、火加減、所要時間、注意点、見極めポイントが入る。AI 良い仕事した。

## 🪤 ハマり集（その他）

### Livewire 4 で `$status` がリザーブ

```
PropertyNotFoundException: Property [$status] not found on component
```

`public string $status = 'idle';` が Blade からアクセスできない。
原因はわからないが Livewire 4 のリザーブネームっぽい。
`$proposalState` にリネームで解決。

### Queue Worker のコードキャッシュ

`Job` のコンストラクタに引数追加したのに反映されない。
Sail の `queue` コンテナが `queue:work` でコードキャッシュしてた。

→ `queue:listen --tries=1` に変えて毎ジョブで子プロセス起動するようにした。開発時はこれが正解。

### 食材マスタの重複

UI からテスト中に手動で「卵」を追加したら slug が `ing-VhjzdH8T` で作られて、後で `egg` slug の seed と被った。
同名・別 ID で 2 個できて在庫照合が壊れた。

→ 重複統合スクリプトで `ing-XXX` → `egg` に付け替え。
→ `createIngredientFromInput` で同名検索を先にやるよう修正。

### main 直接 commit 禁止フック

```
BLOCKED: main ブランチへの直接 commit は禁止です。
feature/fix ブランチを切ってください。
```

ユーザーの hook で main 直叩きが禁止されていた。
`git checkout -b feature/mvp` してから commit。
